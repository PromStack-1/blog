AI 에이전트는 더 이상 대기업만의 기술이 아닙니다. PromStack을 사용하면 **코드 한 줄 없이** 나만의 AI 어시스턴트를 만들 수 있습니다.

## 🤖 AI 에이전트란?

AI 에이전트는 단순한 챗봇을 넘어서 **도구를 사용하고, 정보를 검색하고, 작업을 수행**할 수 있는 지능형 AI입니다.

| 일반 챗봇 | AI 에이전트 |
|----------|------------|
| 텍스트 응답만 가능 | 도구를 사용해 실제 작업 수행 |
| 고정된 지식 | 실시간 정보 검색 |
| 한 번에 하나의 질문 | 복잡한 멀티스텝 작업 |

## 🚀 5분 만에 에이전트 만들기

### Step 1: 에이전트 생성

1. [PromStack](https://promstack.com)에 로그인
2. 좌측 메뉴에서 **에이전트** 클릭
3. **+ 새 에이전트** 버튼 클릭
4. 이름과 설명 입력

### Step 2: AI 모델 선택

PromStack은 다양한 AI 모델을 지원합니다:

| 모델 | 특징 | 추천 용도 |
|------|------|----------|
| **GPT-4o** | 범용성, 빠른 속도 | 일반 대화, 글쓰기 |
| **Claude 3.5 Sonnet** | 긴 컨텍스트, 코딩 | 문서 분석, 개발 |
| **Gemini 1.5 Pro** | 멀티모달, 구글 연동 | 이미지 분석 |

### Step 3: 시스템 프롬프트 작성

시스템 프롬프트는 에이전트의 성격과 역할을 정의합니다.

```markdown
# 역할
당신은 친절한 마케팅 전문가입니다.

# 규칙
- 항상 한국어로 답변합니다
- 질문이 모호하면 명확히 물어봅니다
- 구체적인 예시를 함께 제공합니다

# 전문 분야
- 소셜 미디어 마케팅
- 콘텐츠 전략
- SEO 최적화
```

> 💡 **팁:** 시스템 프롬프트는 명확하고 구체적일수록 좋습니다.

### Step 4: 도구 연동

에이전트에게 능력을 부여하세요!

#### 내장 도구

| 도구 | 기능 |
|------|------|
| 🔍 **웹 검색** | 실시간 정보 검색 |
| 🧮 **계산기** | 수학 연산, 단위 변환 |
| 🌤 **날씨** | 실시간 날씨 정보 |
| 📈 **주식** | 실시간 주가 조회 |
| 🕐 **시간** | 세계 시간대 확인 |

#### 커스텀 API 연동

외부 API를 도구로 연동할 수 있습니다:

```json
{
  "name": "get_product_info",
  "description": "상품 정보를 조회합니다",
  "url": "https://api.mystore.com/products/{id}",
  "method": "GET",
  "headers": {
    "Authorization": "Bearer {{API_KEY}}"
  }
}
```

### Step 5: 테스트 & 배포

1. **미리보기**에서 에이전트 테스트
2. 다양한 질문으로 동작 확인
3. **저장** 후 사용 시작!

## 🎯 실전 에이전트 예시

### 1. 고객 지원 에이전트

```markdown
# 역할
당신은 [회사명]의 고객 지원 담당자입니다.

# 지식
- FAQ 문서 참조
- 반품/교환 정책 숙지

# 규칙
- 친근하고 공손한 말투 사용
- 해결 불가 시 담당자 연결 안내
- 모든 대화 요약 제공
```

**도구:** 웹 검색, 주문 조회 API, 이메일 발송

### 2. 콘텐츠 어시스턴트

```markdown
# 역할
당신은 SEO 전문 콘텐츠 작가입니다.

# 작업
- 블로그 글 작성
- 소셜 미디어 포스트 생성
- 이메일 뉴스레터 초안

# 스타일
- 간결하고 읽기 쉬운 문체
- 적절한 이모지 사용
- CTA 포함
```

**도구:** 웹 검색, 키워드 분석 API

### 3. 개발 어시스턴트

```markdown
# 역할
당신은 시니어 풀스택 개발자입니다.

# 전문 분야
- TypeScript, React, Node.js
- 코드 리뷰 및 최적화
- 버그 디버깅

# 규칙
- 코드는 항상 설명과 함께 제공
- 베스트 프랙티스 권장
- 보안 취약점 경고
```

**도구:** 웹 검색, GitHub API, 문서 검색

## 📊 모니터링 & 개선

### 대화 로그 분석

에이전트의 모든 대화는 자동으로 로깅됩니다:
- 사용자 질문 패턴 파악
- 실패한 응답 분석
- 개선점 도출

### A/B 테스트

다른 시스템 프롬프트로 에이전트를 복제해서 어떤 버전이 더 효과적인지 테스트하세요.

## 💡 에이전트 설계 팁

1. **역할을 명확히** - "당신은 ~입니다"로 시작
2. **제약 조건 설정** - 하지 말아야 할 것 명시
3. **예시 제공** - 원하는 응답 형식 보여주기
4. **도구 최소화** - 꼭 필요한 도구만 연동
5. **지속적 개선** - 로그 분석 후 프롬프트 조정
